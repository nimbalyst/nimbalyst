# Nimbalyst for iOS

Native SwiftUI companion app for [Nimbalyst](https://nimbalyst.com), the AI-native code editor. View and interact with your AI coding sessions on iPhone and iPad, synced in real-time from your Mac.

## Features

- **Real-time session sync** - Sessions appear on your phone as they're created on desktop
- **End-to-end encryption** - All data encrypted with AES-256-GCM; the server never sees plaintext
- **Rich transcript viewer** - Full chat transcripts with code blocks, tool calls, and interactive widgets
- **Session management** - Search sessions, group by time period, create new sessions from mobile
- **iPad split view** - Sidebar navigation on iPad, stack navigation on iPhone
- **Desktop presence** - See when your Mac is online and connected

## Getting Started

### Prerequisites

- macOS with Xcode 16+
- Node.js 20+
- [XcodeGen](https://github.com/yonaskolb/XcodeGen) (`brew install xcodegen`)

### Setup

1. Clone the monorepo and install dependencies:
   ```bash
   git clone https://github.com/nimbalyst/nimbalyst.git
   cd nimbalyst
   npm install
   ```

2. Build the transcript web bundle:
   ```bash
   npm run ios:build:transcript
   ```

3. Generate and open the Xcode project:
   ```bash
   cd packages/ios/NimbalystApp
   xcodegen generate
   open NimbalystApp.xcodeproj
   ```

4. Select a simulator or device and press Run.

### Pairing with Desktop

1. On your Mac, open Nimbalyst and go to **Settings > Session Sync**
2. Enable sync and click **Pair Mobile Device**
3. On the iOS app, scan the QR code
4. Sign in with the same Google account you use on desktop

The QR code contains the server URL and encryption seed. The encryption key is derived locally and never sent to the server.

## Architecture

```
NimbalystNative/     Swift Package with all business logic
  App/               AppState, navigation, lifecycle
  Auth/              Stytch OAuth (Google sign-in)
  Crypto/            AES-256-GCM, PBKDF2 key derivation
  Database/          GRDB (SQLite) with reactive ValueObservation
  Models/            Project, Session, Message, QueuedPrompt
  Sync/              WebSocket sync with CollabV3 server
  Views/             SwiftUI views + WKWebView transcript

NimbalystApp/        Xcode app target (generated by XcodeGen)
src/transcript/      React web bundle for chat transcript (WKWebView)
```

The app connects to the same [CollabV3](../collabv3/) sync server as the desktop app. Session data flows:

```
Desktop (Electron) -> CollabV3 (Cloudflare DO) -> iOS (WebSocket)
```

All data is encrypted client-side before transmission. The server stores only opaque encrypted blobs.

### Project Structure

| Directory | Description |
|-----------|-------------|
| `NimbalystNative/` | Swift Package with all app code (SPM) |
| `NimbalystApp/` | Xcode app target with `project.yml` (XcodeGen) |
| `CryptoCompatibility/` | CommonCrypto bridging for PBKDF2 |
| `src/transcript/` | React transcript loaded in WKWebView |

### Transcript Web Bundle

The chat transcript is rendered by the same React components used in the desktop app, bundled as a standalone IIFE with Vite. The font (Material Symbols) is bundled locally since WKWebView loads from `file://`.

The Xcode pre-build script automatically builds and copies the transcript when you build in Xcode.

## App Store Compliance

### Encryption Export (ITSAppUsesNonExemptEncryption)

The app uses AES-256-GCM encryption (via CryptoKit) for end-to-end encryption of synced data. `ITSAppUsesNonExemptEncryption` is set to `false` in Info.plist because:

1. The encryption uses Apple's built-in CryptoKit framework, not a custom implementation
2. It is used solely for protecting the user's own personal data in transit between their devices
3. It is not used for direct communication between third parties
4. HTTPS/TLS (handled by the OS) is used for all network communication

This qualifies as exempt under US BIS encryption export regulations (Category 5, Part 2, Note 4).

### Privacy Manifest (PrivacyInfo.xcprivacy)

The app declares:
- **UserDefaults** access (reason CA92.1: app-specific preferences)
- **Product interaction** data collection (PostHog analytics, linked to identity)
- **No tracking** (analytics are not used for cross-app advertising)

### Required Legal Links

- Privacy Policy: https://nimbalyst.com/privacy-policy (linked in-app via Settings > Privacy)
- Privacy Policy URL must also be entered in App Store Connect metadata
